What's new in v2

  • Supervisor agent — Claude (via FM API) sits in front of Genie and decides per turn whether to route the question to Genie or answer directly (greetings, meta questions, clarifications).
  • Two conversation IDs, one thread — the app thread_id owns the full turn history in Lakebase; the sticky genie_conv_id is reused across follow-ups ("sort by desc") so Genie keeps its native context. A "New topic" button rotates only the Genie side when you pivot subjects.
  • Memory compression — when the thread exceeds SUMMARY_TURN_THRESHOLD turns or SUMMARY_TOKEN_THRESHOLD tokens, a summarization node folds the older messages into a rolling summary and drops them from state. Summary + recent tail + system prompt is what the LLM sees each turn.
  • Lakebase checkpointer — LangGraph's AsyncPostgresSaver snapshots state after every node. A server restart doesn't lose a conversation. Your app_conversations pointer table lives in the same DB for sidebar listings.
  • ABAC row-level security — Genie runs SQL as the per-dealership SP. UC evaluates dealership_rls() per row: an EXISTS subquery against dealership_principals maps the SP's current_user() to its allowed dealership_id. Adding a new dealership is a single INSERT — no function changes.
